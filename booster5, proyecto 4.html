Posible soluciones a las fallas del soporte

¿Qué provoca errores de detención? Se muestra un error de detención como una pantalla azul que contiene el nombre del controlador defectuoso, como cualquiera de los controladores de ejemplo siguientes: atikmpag.sys igdkmd64.sys nvlddmkm.sys No hay ninguna explicación sencilla para la causa de los errores de detención (también conocidos como errores de pantalla azul o errores de comprobación de errores). Muchos factores diferentes pueden estar implicados. Sin embargo, varios estudios indican que los errores de detención no suelen deberse a componentes de Microsoft Windows. En su lugar, estos errores están relacionados con controladores de hardware que no funcionan correctamente o controladores instalados por software de terceros. Estos controladores incluyen tarjetas de vídeo, tarjetas de red inalámbricas, programas de seguridad, etc. Nuestro análisis de las causas principales de los bloqueos indica que: El 70 % se debe al código de controlador de terceros. El 10 % se debe a problemas de hardware. El 5 % se debe al código de Microsoft. El 15 % tiene causas desconocidas, porque la memoria está demasiado dañada para analizarla.

Nota La causa principal de los errores de detención nunca es un proceso en modo de usuario. Aunque un proceso en modo de usuario (como bloc de notas o Slack) puede desencadenar un error de detención, se limita a exponer el error subyacente, que siempre se encuentra en un controlador, hardware o sistema operativo.

Pasos generales de solución de problemas

Para solucionar problemas de detención de mensajes de error, siga estos pasos generales: Revise el código de error de detención que encuentra en los registros de eventos. Busque en línea los códigos de error de detención específicos para ver si hay problemas conocidos, soluciones o soluciones alternativas para el problema. Asegúrese de instalar las actualizaciones más recientes de Windows, las actualizaciones acumulativas y las actualizaciones de acumulación. Para comprobar el estado de actualización, consulte el historial de actualizaciones adecuado para el sistema. Por ejemplo: Windows 10, versión 21H2 Windows 10, versión 21H1 Windows 10, versión 20H2 Asegúrese de que el BIOS y el firmware están actualizados. Ejecute las pruebas de hardware y memoria pertinentes. Ejecute Examen de seguridad de Microsoft o cualquier otro programa de detección de virus que incluya comprobaciones del MBR para detectar infecciones. Asegúrese de que haya suficiente espacio libre en el disco duro. El requisito exacto varía, pero se recomienda un 10-15 % de espacio libre en disco. Póngase en contacto con el proveedor de hardware o software correspondiente para actualizar los controladores y las aplicaciones en los siguientes escenarios: El mensaje de error indica que un controlador específico está causando el problema. Verá una indicación de un servicio que se está iniciando o deteniendo antes de que se produjera el bloqueo. En esta situación, determine si el comportamiento del servicio es coherente en todas las instancias del bloqueo. Ha realizado cualquier cambio de software o hardware.

Nota Si no hay actualizaciones disponibles de un fabricante específico, se recomienda deshabilitar el servicio relacionado. Para obtener más información, vea Cómo realizar un arranque limpio en Windows. Puede deshabilitar un controlador siguiendo los pasos descritos en Cómo desactivar temporalmente el controlador de filtro de modo kernel en Windows. También puede considerar la opción de revertir los cambios o revertir al último estado de trabajo conocido. Para obtener más información, consulte Reversión de un controlador de dispositivo a una versión anterior.

Recopilación de volcado de memoria

Para configurar el sistema para archivos de volcado de memoria, siga estos pasos: Seleccione el cuadro de búsqueda Barra de tareas, escriba Configuración avanzada del sistema y, a continuación, presione Entrar. En la pestaña Avanzadas del cuadro Propiedades del sistema , seleccione el botón Configuración que aparece en la sección Inicio y recuperación. En la nueva ventana, seleccione la lista desplegable debajo de la opción Escribir información de depuración. Elija Volcado de memoria automático. Seleccione Aceptar. Reinicie el equipo para que la configuración surta efecto. Si el servidor está virtualizado, deshabilite el reinicio automático después de crear el archivo de volcado de memoria. Esta deshabilitación le permite tomar una instantánea del servidor en estado y también si el problema se repite.

Puede usar la herramienta Comprobador de archivos de volcado de memoria (DumpChk) de Microsoft para comprobar que los archivos de volcado de memoria no están dañados o no son válidos. Para obtener más información, vea el siguiente vídeo:

Fallas de Hardware Causa Solucion
1 PC no arranca Fuente de alimentación. Conexiones de la fuente. Memoria RAM mal conectada. Pila de la placa base. Periféricos. Voltaje. Componentes dañados. Darle mantenimiento
2 Pantalla en negro Es un error de hardware porque, en la mayoría de ocasiones, está relacionado con algún componente. También dedicamos un artículo enfocado a la pantalla en negro, ofreciéndoos las soluciones que mejor funcionan con este problema. Muchas veces, es una cuestión de cableado, pero hay ocasiones en las que el fallo proviene de la GPU, de la pila CMOS, discos duros, soporte inadecuado del sistema operativo, etc. Os aconsejamos que echéis un vistazo al tutorial que hicimos porque ahí encontraréis todas las soluciones.
3 No reconoce el disco duro Windows no reconozca el disco duro, ya sea iniciado sesión (si tenemos un varios discos duros) o a la hora de instalarlo. Abrid el menú inicio y buscad «discos» para acceder a «crear y formatear particiones del disco duro«. A continuación, echad un vistazo a los discos duros que tenéis. Si todos están en «azul», significa que tienen un formato y están debidamente conectados. Seguramente, tengáis el disco duro problemático en negro, por lo que no tiene formato y debéis de dárselo. Cuando formateamos un disco duro pierde todos los datos, así que contad con ello.
4 Sobrecalentamiento bajada de rendimiento en el componente sobrecalentado con el objetivo de refrigerarlo. Tener una buena refrigeración en la caja PC a través de un buen circuito de aire. Limpiar el PC es una gran idea para evitar este caso porque los ventiladores podrán refrigerar más. Cuidado con el overclock en CPU y GPU porque podemos exprimir demasiado estos componentes, lo que trae como causa el sobrecalentamiento y, por ende, el thermal throttling. Si no refrigeran bien los componentes, sopesad cambiar la pasta térmica de la GPU y CPU. Dicho esto, antes de cambiar la pasta térmica de la GPU os recomendaría programar una curva de ventilación con MSI Afterburner para que los ventiladores de la misma giren más cuando las temperaturas suban.
5 Memoria RAM: instalación, insuficiente o defectuosa La memoria RAM puede darnos mil errores de harwdare comunes, pero lo cierto es que todos tienen solución, ya sea a coste 0 o comprando algo. Mala instalación. Siempre hay que sacar las pestañas de los slots hacia afuera e introducir los módulos correctamente, sin forzar nada. Cuando los instalamos bien suele oírse un «clic» y las pestañas se retraen como se debe, pero casi siempre tenemos que fijarlas manualmente para que el módulo se quede instalado e inamovible. A veces, no iniciará el PC porque están mal instaladas. Dual channel. Más que un error, se trata de una nula optimización de las memorias y de los slots. Que las placas base vengan con slots de varios colores no es por estética, sino para indicar cómo montar las memorias RAM para hacerlas funcionar en Dual Channel. Instalad las 2 memorias en ranuras del mismo color. Defectuosa. Esto provocará que no arranque el PC y el error aquí es que están rotas, básicamente. Intentad comprar unas nuevas y montarlas para ver si el problema está en las memorias o en la placa.
6 Ranura PCI-Express, ¡no la rompas! Un error de novato es forzar la instalación de la tarjeta gráfica por desconocimiento del funcionamiento de la ranura PCI-Express principal. En todas las placas base modernas, la ranura x16 (la principal) tiene un botón o pulsador que desbloquea la ranura para poder extraer la tarjeta gráfica. En el momento de montar la GPU, no hará falta pulsarlo, pero si queremos extraerla tenemos que pulsarlo porque si no será difícil sacarla. He visto novatos que han roto la ranura PCI-Express por esta tontería, debido a que se ejerce torsión en la ranura y en la PCB de la placa. Cuidado con esto porque podéis romper la placa base.
7 «Clic» en el disco duro Llamado «click of the death«, clic de la muerte, se trata de un fenómeno que se da en HDD o discos duros mecánicos, el cual revela que «queda poco tiempo». Es un síntoma que preavisa de que el disco duro está llegando a su fin Pasar toda la información guardada antes de perderla por completo. Una de las cosas positivas que tienen los HDDs, es que preavisan de la muerte y pueden seguir funcionando, lo que nos da margen para rescatar los archivos. En cambio, los SSDs no avisan y mueren de la noche a la mañana, por lo que tened esto en cuenta.
8 Reinicio inesperado CPU. Simplemente, se queda en negro y se reinicia el equipo. Monitorizad las temperaturas porque todas las CPUs tienen un límite; llegado a éste, se reinician. GPU. Rayas verdes o «crasheo» con ruido en bucle. No tiene por qué ser la temperatura de la GPU, sino que hayáis practicado un mal OC al Memory Clock o Core Clock, así como el voltaje. RAM. Se suele reiniciar sin más. Yo desaconsejo el overclock a la memoria RAM por los pocos beneficios que se consiguen, arriesgando mucho a cambio de poco. Aunque es uno de los errores que se dan en hardware, pero es cierto que está relacionado con el software debido a que el overclock se puede practicar por BIOS o mediante Windows.
9 No funcionan los puertos USB de la caja El problema puede estar originado porque nos hayamos olvidado de conectar el conector de los USB frontales en la placa base. Por un lado, tenéis que encontrar el conector dentro de la cajaEs posible que nuestra placa base no dé tantas salidas a USB frontales. La consecuencia de esto es que habrán puertos USB desactivados por la incompatibilidad entre placa-caja PC. Cuidado con las placas base que dan salidas a USB que la caja PC no tiene, o viceversa. Es posible que el frontal de la caja tenga más salidas USB de las que la placa puede dar, especialmente USB 3.1-3.2., pudiendo estar resguardado en el panel derecho de la misma. Después, hay que encontrar los pines de conexión en la misma placa base.
10 Los ventiladores están parados Sin programación en BIOS o en el sistema operativo. Podemos crear una curva de ventilación en los ventiladores para que giren a ciertas RPM en caso de que se alcancen unas temperaturas «X». Eso se puede hacer desde la BIOS, por lo que acceded a ella para cambiarlo. La otra opción es bajarse un programa para configurar los ventiladores desde Windows. Mi recomendación es que os bajéis el programa de vuestra placa base, el cual os permitirá configurar el disipador y los demás ventiladores de la caja. Sin conectar o mala conexión. Recordad que hay que conectarlos a la placa base, pero no en cualquier conexión de 4-3 pines: en una que ponga «FAN», exceptuando la de «CPU_FAN» porque ahí se conecta el disipador. Avería o demasiado polvo. Por último, pueden estar averiados o simplemente tienen tanto polvo solidificado que impide a las aspas moverse. Para limpiar los ventiladores sucios os recomiendo aire comprimido y alcohol isopropílico para humedecer el polvo, ablandarlo y eliminarlo con más facilidad.
